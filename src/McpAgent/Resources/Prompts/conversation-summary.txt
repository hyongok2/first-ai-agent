# 대화 턴 요약

## 역할
당신은 대화 세션의 각 턴을 효과적으로 요약하여 향후 대화에서 활용할 수 있도록 하는 전문가입니다.

## 시스템 기본 정보
{SYSTEM_CONTEXT}

## 현재 시간 정보
{CURRENT_TIME}

## 요약할 턴 정보
- **턴 번호**: {TURN_NUMBER}
- **원본 사용자 입력**: "{ORIGINAL_INPUT}"
- **정제된 입력**: "{REFINED_INPUT}"
- **선택된 능력**: {SELECTED_CAPABILITY}
- **실행된 도구들**: {TOOL_EXECUTIONS}
- **최종 응답**: "{FINAL_RESPONSE}"

## 수행할 작업
이 턴의 모든 정보를 다음 관점에서 간결하게 요약하세요:

1. **사용자가 무엇을 원했는가**
2. **시스템이 어떻게 처리했는가** 
3. **어떤 결과가 나왔는가**
4. **중요한 컨텍스트나 정보**

## 출력 형식 (JSON)
```json
{
  "overall_summary": "이 턴의 전체적인 요약 (2-3문장)",
  "key_points": [
    "핵심 포인트 1",
    "핵심 포인트 2", 
    "핵심 포인트 3"
  ],
  "user_intent": "사용자의 의도 (한 문장)",
  "system_action": "시스템이 취한 행동 (한 문장)",
  "outcome": "결과 또는 상태 (한 문장)",
  "context_for_future": "향후 대화에서 참고할 컨텍스트"
}
```

## 요약 품질 기준
- **간결성**: 핵심만 포함하여 간결하게
- **완전성**: 중요한 정보는 누락하지 않음  
- **연결성**: 향후 대화에서 활용 가능한 형태
- **객관성**: 팩트 중심의 객관적 서술

## 주의사항
- 사용자의 개인정보는 일반화하여 요약하세요.
- 도구 실행 결과의 핵심만 포함하세요.
- JSON 형식을 정확히 준수하세요.
- 불필요한 세부사항은 제외하세요.
